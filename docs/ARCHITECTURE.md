# Architecture overview

## Monorepo apps
- `apps/webapp` — Telegram Mini App frontend
- `apps/api` — REST API and business logic
- `apps/admin` — moderation and content control panel
- `apps/bot` — Telegram bot entrypoint
- `apps/worker-parser-html` — deterministic HTML parser
- `apps/worker-parser-visual` — OCR and visual parser
- `apps/worker-reconciler` — parser diff and verification worker
- `packages/shared` — shared types and constants

## Core principles
1. Live users may only see verified content.
2. Every question must be traceable to its source.
3. Correct answers may never be generated by AI.
4. Parser disagreements must go to review.

## Data flow
Source snapshot -> parser runs -> parser outputs -> parser diffs -> moderation -> published question
